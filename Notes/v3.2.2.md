# Release Notes v3.2.2

Data de lançamento: 23/03/2026

## 🆕 Resumo

Versão de manutenção com foco no pacote `Tooark.Exceptions`, trazendo novas classes de exceptions alinhadas a códigos HTTP adicionais, suporte a mensagens formatadas via `string.Format` e cobertura de testes ampliada. Também inclui melhorias de documentação do pacote. Não há breaking changes.

---

## Adicionado

- Novas classes de exceptions HTTP no pacote `Tooark.Exceptions`:
  - `ConflictException` (409)
  - `PayloadTooLargeException` (413)
  - `UnsupportedMediaTypeException` (415)
  - `UnprocessableEntityException` (422)
  - `TooManyRequestsException` (429)
- Novos testes unitários dedicados para as novas exceptions adicionadas.

## Modificado

- `TooarkException` recebeu novo construtor com suporte a formatação de mensagens:
  - `TooarkException(string messageFormat, params object[] args)`
- Classes derivadas de exceptions existentes passaram a suportar o mesmo padrão de formatação de mensagem, permitindo criação de erros parametrizados com placeholders (`{0}`, `{1}`, etc.).
- Atualização do `README.md` de `Tooark.Exceptions` com tabela completa de exceptions, exemplos práticos de uso e exemplos com `string.Format`.

## Correções

- Padronização na forma de construção de mensagens e notificações entre as exceptions para cenários de mensagem simples, lista de mensagens, `Notification` e mensagem formatada.
- Ajustes de consistência na documentação para refletir corretamente as exceptions disponíveis e os respectivos status HTTP.

## Testes

- Expansão da suíte de testes de `Tooark.Exceptions` para cobrir:
  - Novas exceptions (`409`, `413`, `415`, `422`, `429`).
  - Construtores com mensagem formatada (`string.Format`) em exceptions novas e existentes.
  - Comportamento de `Message`, `GetErrorMessages()`, `GetNotifications()` e `GetStatusCode()` em todos os formatos de construção.

## Documentação

- `Tooark.Exceptions/README.md`: reformulado e atualizado com visão geral, recursos suportados, lista completa de exceptions e exemplos de uso para cada cenário.

## Impacto

- Sem breaking changes.
- Compatível com .NET 8.0+.
- Melhora a expressividade no lançamento de exceptions por meio de mensagens dinâmicas formatadas.
