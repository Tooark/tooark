# Release Notes v3.3.0

Data de lançamento: 16/04/2026

## 🆕 Resumo

Versão focada na introdução do ecossistema de Mediação do Tooark, com a publicação dos pacotes `Tooark.Mediator` e `Tooark.Mediator.Abstractions`. Esta release adiciona contratos base para CQRS/CQS, implementação concreta de `IMediator`, varredura automática de handlers via DI e estratégia configurável de publicação de notificações. Não há breaking changes nos pacotes já existentes.

---

## Adicionado

### Tooark.Mediator.Abstractions

- Novo pacote de contratos base do padrão Mediator para .NET 8.0.
- Contratos de mensagens:
  - `IRequest<TResponse>` e atalho `IRequest`.
  - `ICommand<TResponse>` e atalho `ICommand`.
  - `IQuery<TResponse>`.
  - `INotify`.
- Contratos de orquestração:
  - `ISender` com `SendAsync<TResponse>(IRequest<TResponse>, CancellationToken)`.
  - `IPublisher` com `PublishAsync(INotify, CancellationToken)`.
  - `IMediator` combinando `ISender` e `IPublisher`.
- Tipo utilitário `Unit` para fluxos sem retorno de valor, incluindo:
  - `Unit.Value`.
  - `Unit.Task`.

### Tooark.Mediator

- Novo pacote com implementação concreta de `IMediator`.
- Suporte completo ao fluxo CQRS/CQS com handlers tipados:
  - `IRequestHandler<TRequest, TResponse>`.
  - `ICommandHandler<TCommand, TResponse>`.
  - `ICommandHandler<TCommand>`.
  - `IQueryHandler<TQuery, TResponse>`.
  - `INotifyHandler<TNotify>`.
- Extensões de injeção de dependência para registro simplificado:
  - `AddTooarkMediator(IServiceCollection, params Assembly[])`.
  - `AddTooarkMediator(IServiceCollection, Action<MediatorOptions>, params Assembly[])`.
- Registro automático de handlers por varredura de assemblies.
- Registro padrão de serviços no container:
  - `IMediator`.
  - `ISender`.
  - `IPublisher`.
- Estratégia configurável para publicação de notificações com `MediatorOptions`:
  - `ENotifyStrategy.ParallelWhenAll` (padrão).
  - `ENotifyStrategy.Sequential`.

## Comportamento e Validações

### SendAsync

- Lança `BadRequestException("Request.Null")` quando a requisição é nula.
- Resolve dinamicamente o handler via `IServiceProvider`.
- Lança `InternalServerErrorException` para cenários de infraestrutura:
  - handler não encontrado;
  - método `HandleAsync` ausente;
  - falha na execução do handler.

### PublishAsync

- Lança `BadRequestException("Notify.Null")` quando a notificação é nula.
- Resolve múltiplos handlers de notificação via `GetServices`.
- Executa handlers em paralelo (`Task.WhenAll`) ou em sequência, conforme estratégia configurada.
- Lança `InternalServerErrorException` quando há inconsistência de método/execução de handlers.
